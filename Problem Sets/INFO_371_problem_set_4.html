My model consists of three convolutional layer with max pooling, a dropout layer with a dropout rate of 0.25, a flatten layer, and two dense layers, one with relu activiation, and the other with softmax activation. The first convolutional layer has a smaller kernal size than the following two convolutional layers, and I squeezed in a leaky ReLU and batch normalization layer before the max pooling layer.

Changes were made based on the first 3 epochs of fitting the data - if it did not reach the 85% threshold within that, I would either change the dropout rate, move where Leaky ReLU and BatchNormalization were in the model, change the number of nodes in the relu activated dense layer, or change the kernal size of the first convolutional layer.

The reason behind the added models was of help with the linked TensorFlow example, the in-class example, the lab, and also the diagram shown in class. It helped to visualize how the data was flowing throughout the model in order to fine tune it.
